# Future Features: Quran Recitation Tester (Phase 2)

These features are planned for future development to enhance the mobile experience and user motivation.

### 1. Mobile Haptic Feedback
- **Description**: Provide subtle vibrations on mistakes or milestone completions.
- **Benefit**: Physical cues that don't require looking at the screen.

### 2. Session Persistence
- **Description**: Use `localStorage` to save the last selected Surah, Ayah range, and settings.
- **Benefit**: Resume exactly where you left off after refreshing.

### 3. Recitation Statistics & Motivation
- **Description**: Track daily Ayah completion counts and "streaks".
- **Benefit**: Gamification to encourage daily practice and memorization progress.

### 4. Offline PWA Support
- **Description**: Implement a Service Worker and Web Manifest.
- **Benefit**: Run the app completely offline and install it like a native app on iOS/Android.

### 5. Multi-Reciter Support
- **Description**: Allow users to switch between different reference reciters (e.g., Al-Afasy, Al-Husary).
- **Benefit**: Practice with different styles and pronunciations.

### 6. Visual Refinement
- **Description**: Dark/Light mode toggle, smoother Ayah-to-Ayah transitions, and Tajweed-colored text.
- **Benefit**: Reduced eye strain during long sessions and better visual guidance.
